### 通用编码规范

实现任何代码前，先阅读这些文件了解项目：
- 构建/依赖文件（如 `package.json`、`pyproject.toml`、`pom.xml`、`Makefile`）
- 现有源代码 — 命名规范、模块结构、使用的模式
- 测试文件 — 了解使用的测试框架和模式

**实现流程**：
1. **理解**目标 feature 的验收标准
2. **实现**所需的代码改动，遵循项目现有规范
3. **测试** — 运行相关测试
4. **更新** {{workspace}}/task_list.json：将该 feature 的 `passes` 设为 `true`
5. **提交**：`git add -A && git commit -m "<feature-id>: <简短描述>"`

**通用最佳实践**：
- 遵循现有的代码风格、命名规范和模式
- 函数职责单一 — 一个函数做一件事
- 写自解释的代码，使用清晰的变量名和函数名
- 显式处理错误 — 不要静默吞掉异常
- 使用项目既定的日志机制（不要用 print/console.log）
- 始终保持工作空间处于干净、可运行的状态
